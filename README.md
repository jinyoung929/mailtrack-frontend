# 메일트랙 (MailTrack)

AI 기반 이메일 보안 분석 서비스입니다. Gmail API로 실제 메일 데이터를 가져오고, Gemini 기반 분석을 통해 보안 등급 분류, 개인정보·다크데이터 탐지, 일정 추출, 사용자 키워드 기반 필터링을 수행합니다.

- 배포 링크: https://illustrious-cupcake-8c5538.netlify.app
- 백엔드 API: https://mailtrack-backend.onrender.com

## Project Context

본 저장소는 2023년 팀 프로젝트를 바탕으로, 2026년 포트폴리오 공개를 위해 개인적으로 재구현한 버전입니다. 기존 프로젝트에서 부족했던 인증, 백엔드 라우터, Gmail 실연동, 예외처리, AI 판단 기준을 직접 설계·구현했습니다.

| 영역 | 원본 | 메일트랙 개인 구현 |
| --- | --- | --- |
| 메일 소스 | Mock 데이터만 사용 | Gmail API 실연동 |
| 인증 | 없음 | Google OAuth 2.0 직접 구현 |
| 보안 판단 | AI 자유 판단 | 명시적 탐지 기준 + 사용자 키워드 반영 |
| 일정 기능 | 없음 | 메일 본문에서 일정 자동 추출 |
| 백엔드 라우터 | 미구현 | analyze / mails / keywords / auth 구현 |
| 에러 처리 | 없음 | API 쿼터 초과 등 예외 상황 fallback |
| UI | 기본 디자인 시스템 | 전체 리디자인 |

## My Role

- Google OAuth 2.0 및 Gmail API 연동
- FastAPI 기반 analyze / mails / keywords / auth 라우터 구현
- Gemini 기반 메일 요약, 보안 등급 분류, 다크데이터 탐지 로직 구현
- 개인정보, 민감정보, 광고성 메일, 의심 첨부파일 등 탐지 기준 설계
- 사용자 키워드 기반 스마트 필터 및 분석 기준 반영
- API 쿼터 초과 등 예외 상황에 대한 fallback 처리
- React/TypeScript 기반 UI 구조 재설계 및 배포 환경 구성

## Project Timeline

- 원 프로젝트: 2023.03 - 2023.05
- 개인 재구현 및 포트폴리오 정리: 2026.06

## Why This Project Matters

메일함에는 개인정보, 민감정보, 광고성 메일, 의심 첨부파일 등 업무상 관리가 필요한 비정형 데이터가 섞여 있습니다. 메일트랙은 Gmail 데이터를 직접 연동해 위험 신호를 구조화하고, AI 분석 결과를 사용자가 검토할 수 있는 기준과 함께 보여주는 데 초점을 두었습니다.

이 프로젝트는 단순 AI 데모가 아니라 실제 외부 API 인증, 데이터 수집, 분석 기준 설계, 예외처리, 배포까지 연결한 서비스 구현 경험입니다.

## 다크 데이터 탐지 기준

1. 광고성 메일: 제목에 (광고) 표시 또는 명백한 마케팅 목적
2. 발신자 회신 불가 주소: no-reply, noreply 등
3. 개인정보 포함: 주민번호, 계좌번호, 카드번호, 전화번호, 주소 패턴
4. 민감 의료정보: 진단명, 처방전, 검사결과 등
5. 첨부파일 언급: 압축파일(.zip .rar), 매크로 문서(.xlsm .docm) 등 의심 확장자
6. 중복/불필요 데이터: 동일 내용 반복, 만료된 이벤트/공지

## 주요 기능

| 기능 | 설명 |
| --- | --- |
| Gmail 실연동 | Google OAuth 2.0으로 로그인 후 Gmail API로 실제 받은 메일함 분석 |
| AI 메일 분석 | Gemini 2.5 Flash로 보안 등급 분류, 요약, 다크데이터 탐지 |
| 보안 센터 | 위험/주의/안전 등급별 메일 분류 및 분석 근거 표시 |
| 다크데이터 탐지 | 개인정보·민감정보·광고성 메일·의심 첨부파일 등 6가지 기준 탐지 |
| 일정 자동 추출 | 메일 본문에서 날짜·시간·장소를 인식해 캘린더 카드로 시각화 |
| 스마트 필터 | 사용자 정의 스팸 키워드 등록 후 분석 시 자동 반영 |

## 기술 스택

- Frontend: React 19, TypeScript, Tailwind CSS v4, Vite
- Backend: FastAPI, Gemini API
- DB: Supabase(PostgreSQL)
- 배포: Netlify(FE), Render(BE)
